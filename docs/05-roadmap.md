# 开发路线图

## 阶段一: 原型验证 (当前目标)

- [ ] 创建项目基础结构，Ant 构建脚本
- [ ] 实现基础 HTTP 请求客户端
- [ ] 简单 JSON 请求/响应解析
- [ ] 基本文字输入界面 (TextBox + Command)
- [ ] 在 W995 模拟器中测试联网
- [ ] 真机安装测试 basic ping 请求

**完成目标**: 能从 W995 成功发送文字到 OpenClaw 服务器并收到响应显示在手机上。

## 阶段二: 功能完善

- [ ] 历史消息记录
- [ ] 可配置服务器地址 (不用硬编码)
- [ ] 会话管理 (保持会话 ID)
- [ ] 滚动显示长文本响应
- [ ] 错误处理和重试机制
- [ ] 配置文件保存到存储卡

## 阶段三: 服务端适配

- [ ] OpenClaw 服务端 API 端点
- [ ] 身份认证 (简单 API key 认证)
- [ ] 请求限流，防止滥用
- [ ] 可选: 语音合成返回音频文件，W995 播放
- [ ] 文档: Nginx 反向代理配置示例

## 阶段四: 体验优化

- [ ] T9 输入联想优化
- [ ] 字体大小适配
- [ ] 深色/浅色主题
- [ ] 快捷键支持 (比如 OK 发送，C 清除)
- [ ] 预编译打包好 JAD/JAR 供下载

## 阶段五: 扩展功能 (可选)

想一想还能做什么:

- [ ] 天气查询客户端 (调用天气 API 显示在功能机上)
- [ ] 日历同步 (手机看日程，服务器创建事件)
- [ ] RSS 新闻阅读
- [ ] 蓝牙键盘支持
- [ ] 图片识别: 拍照后发送到服务器识别，返回描述 (W995 有相机)
- [ ] 语音输入: 录音后发回服务器转文字再处理

## 长期想法

如果这个项目成了，可以考虑:

- 支持更多 J2ME 机型
- 通用代理协议，方便连接各种后端
- 可以做一个超轻量级 "AI 功能机" 产品
