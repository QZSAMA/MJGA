# 可行性分析与挑战解决

## 整体可行性评估

**✅ 完全可行！**

索尼爱立信 W995 具备我们项目需要的所有条件：

- [x] 内置 WiFi 模块，系统层面支持连接现代路由器（WPA2）
- [x] MIDP 2.0 提供标准 `HttpConnection` HTTP API，可以直接发请求
- [x] 1-2MB 可用堆内存足够运行一个简单文本聊天 MIDlet
- [x] 物理 T9 键盘输入，体验比触屏虚拟键盘更好
- [x] 待机时间长，适合作为 "AI 终端" 使用

## 主要挑战与解决方案

### 1. TLS 版本不兼容（最大问题）

**问题描述**:
- W995 的 J2ME 实现只支持 TLS 1.0
- 现代 HTTPS 服务器基本都禁用了 TLS 1.0
- 直接 HTTPS 连接 OpenClaw 服务器会握手失败

**解决方案**: **反向代理降级**

在你的内网或者一台 VPS 上放一个 Nginx 反向代理：

```nginx
server {
    listen 80;
    server_name your-proxy-domain.com;

    location /openclaw/ {
        proxy_pass https://your-openclaw-server.com/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

然后 J2ME 客户端访问 `http://your-proxy-domain.com/openclaw/`，反向代理用 HTTPS 转发。手机端不需要加密（内网或专用网络环境安全），服务器端仍然用 HTTPS，两全其美。

**替代方案**: 如果你的 OpenClaw 已经在公网，也可以直接给一个 HTTP 端口只允许你代理访问，不对外开放。

---

### 2. 堆内存有限

**问题描述**:
- W995 只有大约 1.5MB 堆内存可用
- 如果包含 JSON 库和 UI 代码，需要控制大小

**解决方案**:
- **精简原则**: 客户端只做输入、发送、显示，不做任何复杂处理
- 使用非常小巧的 JSON 库（< 10KB），或者针对我们的简单格式手工解析
- 分段显示长响应，不要一次性把整个响应加载到内存
- 及时关闭输入输出流，帮助 GC 回收

根据估算，一个简单 MIDlet 带 JSON 解析总共才 100KB 以内，远远小于限制。

---

### 3. 开发工具难找

**问题描述**:
- Oracle 已经从官网撤下所有 Java ME 下载链接
- 新版 NetBeans 不支持 Java ME 插件

**解决方案**:
- 互联网档案馆（Archive.org）保存了所有需要的安装包
- 本文档整理了所有下载链接，参见 [J2ME 开发环境搭建](./02-j2me-setup.md)
- 也可以用轻量级 Ant 构建，不依赖大型 IDE

---

### 4. 中文显示问题

**问题描述**:
- 部分老式功能机中文字体不全，或者编码问题

**解决方案**:
- W995 港版/亚太版原生支持中文显示和输入
- 如果是欧版，可以刷带中文的固件
- 编码使用 UTF-8，多数 W995 都支持

---

### 5. 输入速度问题

**问题描述**:
- T9 键盘输入中文比触屏慢

**解决方案**:
- 本来就是主打极简，慢速输入更有助于思考
- T9 联想输入其实熟能生巧，发短消息问题不大
- 如果需要长文本，可以分段发送
- 如果追求速度，也可以外接蓝牙键盘（W995 支持蓝牙 HID）

---

### 6. 应用安装问题

**问题描述**:
- 现在没有办法直接通过应用商店安装
- 部分手机要求签名才能安装

**解决方案**:
- 通过存储卡传输安装包直接安装
- 个人使用不需要去除权限提示，每次点确认就行
- 也可以自签名，去掉部分提示

---

### 7. 蜂窝网络兼容性

**问题描述**:
- 国内运营商陆续退网 2G/3G

**解决方案**:
- 主要使用场景是 WiFi 连接，在家用不需要蜂窝
- 如果需要外出，确认你所在地区 3G 信号仍然可用
- 或者使用支持 3G 的物联网卡

## 风险等级汇总

| 挑战 | 风险等级 | 解决方案就绪 |
|------|----------|--------------|
| TLS 版本不兼容 | 高 | ✅ 反向代理方案可用 |
| 内存限制 | 中 | ✅ 代码精简可以解决 |
| 开发工具难找 | 中 | ✅ Archive.org 有存档 |
| 中文显示 | 低 | ✅ 亚太版原生支持 |
| 输入速度 | 低 | 产品设计接受慢速输入 |
| 退网问题 | 低 | 主打 WiFi 使用场景 |

## 结论

技术上没有不可逾越的障碍，主要工作就是写出 MIDlet 客户端和适配 OpenClaw 服务端接口。这个项目主要是好玩，探索老硬件新用法，技术挑战都有明确解决方案。
